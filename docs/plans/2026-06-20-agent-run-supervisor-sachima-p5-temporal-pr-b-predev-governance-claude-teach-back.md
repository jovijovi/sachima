## Critical flows and failure cases I must design for later

For each, I restate the required behavior and the stable codes/markers I must preserve — not a design.

- **Happy path.** Enable flag on and exact approval token present → caller supplies a `temporalio` client → `P5TemporalStepExecutor.execute(request, *, role_binding, resolved_inputs)` → `P5TemporalControlSurface` dispatches `start` → `StepWorkflow` runs deterministically under a workflow id bound to caller refs + step idempotency material → `step_activity` runs a **controlled-deterministic** body (no real `acpx`/agent) → artifacts surface as **claim-check refs only** → query returns a sanitized snapshot → `close`/terminalize → `StepExecutionOutcome` with claim-check refs. **No business verdict is inferred from executor success.**
- **Duplicate start — identical.** Returns an idempotent/replayed projection and **launches nothing twice**; reconciled via the deterministic workflow id.
- **Duplicate start — divergent.** **Fails closed** with `runtime_idempotency_conflict` or an `invalid_start_payload`-style stable code. I must bind reconciliation to sanitized observable payloads such that **public digests are not treated as trust boundaries** (PRD open question #4).
- **Restart/recovery.** Reattaches **by workflow id**, **never auto-relaunches uncertain work**; Worker restart/replay/recovery occurs **without duplicate execution**; `recover` returns a no-throw sanitized result.
- **Cancellation WATCH.** WP3b **active-run cancellation WATCH is preserved**; `cancel` returns a no-throw sanitized result and must **not overclaim cancellation success** (open question #5). `active_run_watch` / cancellation-recovery markers are allowed in history.
- **Event-history leak.** Both **SCAN 1 (JSON projection of durable records/query snapshots/evidence)** and **SCAN 2 (serialized real event-history bytes scanned as JSON and as raw bytes)** must contain only allowed fields/values. Any hit is a **blocker → `runtime_history_leak_detected`**. Allowed vs forbidden:

  ```text
  ALLOWED: stable codes; mode/phase; run/workflow/activity/step refs; role keys;
  claim-check refs + sha256 digests; artifact/evidence refs + digests;
  counts/indices/versions; sanitized lease_id/lease_epoch; caller verdict code;
  cancellation/recovery markers incl. active_run_watch
  FORBIDDEN: raw prompt/context/model output; raw acpx/ACP/agent stdout;
  exception text/tracebacks; PIDs/thread ids/host names; platform private ids/
  card JSON/message ids; media bytes/private paths; tokens/credentials/secrets/
  connection strings/signed URLs; delivery payloads/IM bodies
  ```
- **Disabled / mismatched token.** Flag off or token missing → **no Temporal call is made**; caller stays on the local/offline baseline. Failure shapes: `runtime_disabled`, `runtime_approval_mismatch`, `runtime_precondition_unmet`.
- **Gateway-boundary violation.** A **static Gateway-boundary test is required** and must **fail** if any Gateway/runtime message path imports or references the Worker/service-lifecycle surface.
