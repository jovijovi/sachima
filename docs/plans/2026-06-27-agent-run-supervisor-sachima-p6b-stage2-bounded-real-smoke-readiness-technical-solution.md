# P6-B Stage-2 bounded real-smoke readiness — No-code technical solution

Date: 2026-06-27
Status: Docs-only technical solution. No real smoke approval.

## 1. Architecture verdict

`PASS_WITH_WATCH` for the readiness/governance packet; `BLOCKED` for real smoke execution.

The Stage-1 source bridge is the right shape: it is a thin StepExecutor bridge over existing controlled local exec walls, default-off, read-only, claim-check-only, and non-runnable with committed null-binary roles. The next problem is not source architecture; it is execution readiness.

## 2. Candidate execution path, when later approved

```text
caller-owned operator harness, never Gateway:
  1. Validate P6/P6-B exact approval token and operator gate.
  2. Validate one workflow with exactly one planning/report step.
  3. Validate read-only role binding and role overlay digest.
  4. Validate pinned local acpx binary path/version/sha and reject all package runners.
  5. Validate preflight/lease/state-version record.
  6. Acquire WP4 claim before executor call.
  7. Acquire controlled-exec claim before supervisor boundary.
  8. Materialize deterministic P6-B prompt after claim only.
  9. Invoke exactly one read-only one-shot agent process.
  10. Write report bytes to an out-of-repo sink.
  11. Return exactly one sanitized ArtifactRef to WP4/P6.
  12. Post-verify no relaunch/replay, no leaks, no writes, no live surfaces, clean process state.
```

This branch does not run those steps. It records what must be true before the user can approve them.

## 3. Exact command class

The later command must be generated by a host-local operator harness, not typed ad hoc. It must encode:

```text
absolute pinned acpx binary + sha256 + version
one-shot exec/session strategy only
read-only role key + adapter pin
repo/workdir identity
single workflow id + single step id + output contract
max turns + max seconds
out-of-repo workdir + evidence root
no shell, no npx/npm/pnpm/yarn/bun/bunx/node/sh/bash/env trampoline
```

The command line must be argv-list/no-shell. If the implementation path cannot expose the final argv safely, the run must record a sanitized command-shape projection with binary digest, adapter, role key, prompt ref, and bounds rather than raw shell text.

## 4. Role and prompt rules

Allowed role candidates are existing read-only controlled roles only:

- `sachima.codex.primary_reviewer`
- `sachima.claude.read_only_reviewer`

Any later host-local role overlay must keep write/execute/terminal/delete/move/fetch/switch_mode/other permissions false. It may set only a verified local binary path and runtime-local fields needed for execution. The committed role remains portable and non-runnable.

Prompt materialization must use `materialize_p6b_planning_report_prompt`. Raw prompt text must never be stored in durable claim state, history, query projection, roadmap docs, or user-visible evidence. The fixture mirror must match the builder byte-for-byte at the final head.

## 5. Evidence and artifact sink rules

The artifact sink must be caller-owned and out-of-repo. It may persist report bytes only outside the tracked worktree and must return exactly one sanitized `ArtifactRef` with allowed fields only. The final evidence bundle must include:

- runner provenance projection: path class, version, binary sha256, role overlay digest;
- prompt ref and prompt digest, not raw prompt;
- workflow/step/output-contract refs;
- pre-launch claim ids and idempotency refs;
- execution counts proving exactly one launch;
- replay/recover projection proving no second launch;
- artifact ref and digest only;
- redaction/no-leak scan result;
- tracked worktree cleanliness;
- process cleanup proof;
- explicit no Gateway/Feishu/live/production/delivery proof.

## 6. Crash / recovery / no-relaunch gate

Stage-2 cannot request real smoke until one of the following is proven:

### Option A — Durable claim-store path

A separately approved implementation attaches a cross-process durable claim store and proves:

- claim survives process restart;
- query/recover sees resident in-progress/terminal state after restart;
- identical replay does not launch;
- conflicting replay fails closed;
- crash after claim but before finalize does not auto-relaunch without operator gate.

This option is implementation work and is **not** approved by this docs-only branch.

### Option B — Fail-closed host-local proof

Without a new durable store, the host-local DoR must prove the current in-process path fails closed after crash/restart and cannot relaunch from stale input. If it cannot prove that, real smoke remains blocked.

## 7. Kill criteria

Stop before real smoke if any of these hold:

- exact runner path/version/sha is missing or uses a package runner;
- role overlay digest is missing or enables write/execute/fetch/terminal-like capabilities;
- prompt fixture/digest mismatch;
- artifact sink writes inside the repo or returns zero/multiple/unsafe refs;
- any host-local proof would require Gateway/Feishu/live/production config/service restart;
- crash/recovery proof cannot distinguish reattach from relaunch;
- any raw prompt/output/path/token/card/media/platform id appears in durable/user-visible evidence;
- worktree is dirty before launch.

## 8. Next gate recommendation

After this readiness PR, request **host-local DoR / crash-no-relaunch proof approval**, not real smoke execution. Only after that DoR is green should Hermes ask the user for a single real-smoke execution approval with all values filled in.
